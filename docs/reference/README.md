# Reference Docs

This directory is reserved for stable reference material such as wire formats, error vocabularies, and long-lived compatibility notes.

Reference documents should only live here once the corresponding behavior is considered stable enough to serve as a contract.

Current reference documents:

- `zig_coding_rules.md` for the rules index that links to the task-specific Zig rule documents.
- `zig_coding_rules/design_and_safety.md` for implementation shape, assertions, memory, and error handling.
- `zig_coding_rules/performance.md` for performance planning and hot-path guidance.
- `zig_coding_rules/api_and_style.md` for naming, comments, API shape, and code ergonomics.
- `zig_coding_rules/repo_workflow.md` for structure, planning, and automation rules.
- `zig_coding_rules/testing_and_docs.md` for test strategy and documentation layering.

Related documentation:

- `docs/design/` for stable design notes.
- `docs/decisions/` for boundary and ownership decisions.
- `docs/plans/` for active work and completed implementation history.
