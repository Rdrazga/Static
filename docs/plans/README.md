# Plans

This directory is the working ledger for non-trivial repository changes.

- Start with `docs/plans/active/README.md` for the current in-flight work.
- Use `docs/plans/active/workspace_operations.md` for the workspace-wide
  priority order and sequencing.
- `docs/plans/active/` holds in-flight implementation plans.
- `docs/plans/active/README.md` is the entry point for the active workspace and
  package plans.
- `docs/plans/active/packages/` holds only package plans with current
  unfinished work.
- Monitor-only or trigger-only follow-up belongs in `docs/plans/completed/`,
  not in `docs/plans/active/packages/`.
- Active tasks should be written as ordered SMART steps: specific surface,
  measurable completion signal, attainable bounded scope, relevant package or
  workspace goal, and ordering-based timing instead of date promises.
- `docs/plans/completed/` holds completed plans, reviews, and remediation records.
- `docs/sketches/` holds exploratory notes that are not yet stable enough to treat as an execution plan.

Use a plan when the work spans multiple files, introduces new workflow surface area, or changes repository structure.
