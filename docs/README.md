# Documentation Map

This directory is the top-level map for the workspace documentation tree.

## Folders

- `architecture.md`: current package-layer map and dependency direction.
- `design/`: stable design documents that outlive a single implementation pass.
- `decisions/`: short decision records for boundary and ownership calls.
- `plans/`: active implementation plans plus completed plan and review history.
- `reference/`: durable reference material and coding rules.
- `sketches/`: exploratory notes that have not yet become active plans.

## Navigation rules

- Use `docs/plans/active/workspace_operations.md` for the current active queue.
- Use `docs/plans/completed/` for finished plans, reviews, and remediation
  records.
- Use `docs/sketches/archive/` for historical sketches that are no longer part
  of the live sketch surface.
- Keep stable contracts in `reference/` or `design/`, not in active plans.
